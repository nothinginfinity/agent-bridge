# Message OS v0.2 — Agent Bridge Sync

**status:** next build target
**worker:** `message-os-mcp`
**database:** `message-os-db`

## Goal

Connect Agent Bridge messages to Message OS notifications.

Current v0.1 works:

```txt
send_human_message
→ D1 message
→ D1 notification
→ check_human_inbox
→ propose_notification_frame
→ record_frame_choice
→ read_human_message
→ archive_message_to_drivemind
```

v0.2 adds:

```txt
Agent Bridge inbox/outbox file
→ parse message blocks
→ ingest as Message OS message
→ create Message OS notification
→ surface with same UI-frame/action flow
```

## New tools

```txt
ingest_agent_bridge_message
sync_agent_bridge_file
check_agent_bridge_notifications
reply_to_agent_bridge_message
archive_agent_bridge_message
```

## Required Worker additions

### `ingest_agent_bridge_message`

Accept a parsed Agent Bridge message and store it in Message OS.

```json
{
  "source_file": "chatgpt/inbox.md",
  "bridge_message_id": "MSG-C-G-002",
  "from_agent": "claude",
  "to_agent": "chatgpt",
  "project": "agent-bridge",
  "type": "status",
  "priority": "high",
  "subject": "cloudflare deploy complete",
  "body": "...",
  "requires": "cloudflare"
}
```

### `sync_agent_bridge_file`

Read or receive a bridge markdown file, parse message blocks, and ingest only new message IDs.

MVP modes:

```txt
content mode: caller passes markdown content
GitHub mode: Worker reads file with GITHUB_TOKEN
```

### `check_agent_bridge_notifications`

Alias/filter around existing notification query for agent-bridge source events.

### `reply_to_agent_bridge_message`

Create a reply through either Message OS storage or Agent Bridge Comms MCP/GitHub bridge once connected.

## Required bindings

For content-mode MVP:

```txt
DB only
```

For GitHub-mode sync:

```txt
GITHUB_TOKEN
GITHUB_OWNER = nothinginfinity
GITHUB_REPO = agent-bridge
GITHUB_BRANCH = main
```

## Safety/defaults

- Do not auto-open full message bodies.
- Do not mark bridge messages read automatically.
- Ingest duplicate bridge message IDs only once.
- Preserve source metadata.
- Use the existing UI-frame approval flow before reading/replying/archiving.
