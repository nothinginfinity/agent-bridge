## Overview

`afo-buttons-mcp` is a Cloudflare Worker MCP that **generates, audits, and repairs button systems** for other Cloudflare Worker UIs.

It does not serve buttons to end users. It is a factory tool. The final output is HTML + JavaScript snippets that get assembled into a deployable Worker page by `afo-ui-composer-mcp` (or applied directly by an agent).

**Why buttons first:** Buttons are the most common point of silent failure in the AFO dashboard ecosystem. A Worker can deploy successfully, the API can work, and the page can visually load — but buttons can silently fail because they have no handler, no loading state, or no error target. This MCP makes every button a declared, verifiable contract item.

## The Button Contract

Every button generated by this MCP is a **declared action**, not raw HTML.

### Button Spec Schema (input)

```json
{
  "id": "loadSource",
  "label": "Load source",
  "variant": "primary",
  "action": "fetch",
  "endpoint": "/api/source/:worker",
  "method": "GET",
  "requires": ["admin_passcode", "worker_id"],
  "loading_label": "Loading…",
  "success_target": "sourceOut",
  "error_target": "sourceStatus",
  "confirm": false,
  "admin_only": false,
  "copy_value": null
}
```

### Button Spec Fields

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | string | ✅ | Unique DOM id. Used as `data-btn-id` for audit hooks. |
| `label` | string | ✅ | Button label text. |
| `variant` | enum | ✅ | `primary` \| `secondary` \| `ghost` \| `danger` \| `copy` \| `admin` |
| `action` | enum | ✅ | `fetch` \| `copy` \| `navigate` \| `toggle` \| `submit` \| `noop` |
| `endpoint` | string | fetch only | Target URL or path. Supports `:param` tokens. |
| `method` | string | fetch only | `GET` \| `POST` \| `DELETE`. Default: `GET`. |
| `requires` | string[] | optional | Variables that must be resolved before the action fires. Button is disabled until all are present. |
| `loading_label` | string | optional | Label shown during fetch. Default: `"Loading…"` |
| `success_target` | string | optional | DOM id to write success response into. |
| `error_target` | string | optional | DOM id to write error message into. |
| `confirm` | boolean | optional | Show native confirm() before action. Default: false. |
| `admin_only` | boolean | optional | Button hidden unless `admin_passcode` is resolved. |
| `copy_value` | string | copy only | Static string or `{variable}` token to copy to clipboard. |

---

## Button Variants

| Variant | Use Case | Default Style |
|---|---|---|
| `primary` | Main CTA per section (max 1 per group) | Filled accent background |
| `secondary` | Secondary actions | Outlined, same accent |
| `ghost` | Tertiary / de-emphasized | Text only, hover background |
| `danger` | Destructive actions | Red/error color, requires `confirm: true` |
| `copy` | Clipboard copy | Icon + label, shows ✓ on success |
| `admin` | Jared-only admin controls | Hidden until passcode resolved |

## MCP Tools

### `generate_button`

Generates a single contract-bound button.

**Input:**
```json
{
  "spec": { ...ButtonSpec }
}
```

**Output:**
```json
{
  "id": "loadSource",
  "html": "<button data-btn-id=\"loadSource\" ...>Load source</button>",
  "js": "document.querySelector('[data-btn-id=\"loadSource\"]').addEventListener(...)",
  "contract_entry": { ...ButtonSpec },
  "requires_vars": ["admin_passcode", "worker_id"]
}
```

### `audit_buttons`

Validates all declared buttons against a UI contract. Designed to run pre-deploy and post-deploy.

**Input:**
```json
{
  "page_source": "<full HTML string of the Worker page>",
  "contract": {
    "buttons": [ ...ButtonSpec[] ]
  }
}
```

**Output:**
```json
{
  "pass": false,
  "total": 4,
  "passed": 3,
  "failed": 1,
  "checks": [
    { "id": "refresh",     "exists": true,  "has_handler": true,  "has_loading": true,  "has_error_target": true,  "pass": true },
    { "id": "copyAll",     "exists": true,  "has_handler": true,  "has_loading": false, "has_error_target": false, "pass": false,
      "issues": ["missing_loading_state", "missing_error_target"] },
    { "id": "loadSource",  "exists": true,  "has_handler": true,  "has_loading": true,  "has_error_target": true,  "pass": true },
    { "id": "copySource",  "exists": true,  "has_handler": true,  "has_loading": false, "has_error_target": false, "pass": true }
  ],
  "repair_plan": [
    {
      "id": "copyAll",
      "issues": ["missing_loading_state", "missing_error_target"],
      "action": "call repair_button",
      "note": "Add loading label and wire error_target to a status element."
    }
  ]
}
```

### `repair_button`

Returns a corrected HTML + JS snippet for a button that failed audit.

**Input:**
```json
{
  "id": "copyAll",
  "original_spec": { ...ButtonSpec },
  "issues": ["missing_loading_state", "missing_error_target"],
  "page_source": "<optional: full page HTML for context>"
}
```

**Output:**
```json
{
  "id": "copyAll",
  "html": "<button data-btn-id=\"copyAll\" ...>Copy all</button>",
  "js": "...",
  "diff_notes": [
    "Added data-loading-label attribute.",
    "Added error write to #copyAllStatus. Create that element if missing."
  ]
}
```

### `generate_copy_button`

Shorthand for generating a clipboard copy button with ✓ success feedback.

**Input:**
```json
{
  "id": "copyWorkerUrl",
  "label": "Copy URL",
  "copy_value": "{worker_url}",
  "success_label": "Copied!",

```

**Output:** HTML + JS snippet with clipboard write, success state, and auto-reset.

### `generate_admin_button`

Shorthand for generating an admin-gated button that stays hidden until `admin_passcode` is resolved.

**Input:**
```json
{
  "id": "deleteWorker",
  "label": "Delete worker",
  "variant": "danger",
  "action": "fetch",
  "endpoint": "/api/delete/:worker_id",
  "method": "DELETE",
  "confirm": true,
  "admin_only": true
}
```

**Output:** HTML + JS with `hidden` attribute toggled by passcode resolution event, confirm guard, and fetch handler.

## Generated Code Contracts

Every generated button and handler follows these rules:

### HTML Contract
- Button has `data-btn-id="{id}"` — used as the audit hook
- Button has `data-action="{action}"` — machine-readable action type
- Button has `data-endpoint="{endpoint}"` if action is `fetch`
- Button has `disabled` until all `requires` vars are resolved
- Button has `data-loading-label="{loading_label}"` if action is `fetch`

```html
<button
  data-btn-id="loadSource"
  data-action="fetch"
  data-endpoint="/api/source/:worker"
  data-loading-label="Loading…"
  disabled
  class="btn btn-primary"
>
  Load source
</button>
```

### JS Handler Contract
Every fetch button handler must:
1. Set `disabled = true` and swap label to `loading_label` on click
2. Resolve `:param` tokens from the state registry before firing
3. Write response to `success_target` element on success
4. Write error message to `error_target` element on failure
5. Restore button label and `disabled = false` after response

```js
// Generated handler — loadSource
(function() {
  const btn = document.querySelector('[data-btn-id="loadSource"]');
  if (!btn) return;

  btn.addEventListener('click', async () => {
    const workerId = window.__afoState?.worker_id;
    const passcode = window.__afoState?.admin_passcode;
    if (!workerId || !passcode) return;

    const originalLabel = btn.textContent;
    btn.disabled = true;
    btn.textContent = btn.dataset.loadingLabel || 'Loading…';

    try {
      const res = await fetch(`/api/source/${workerId}`, {
        headers: { 'X-Admin-Passcode': passcode }
      });
      const text = await res.text();
      const target = document.getElementById('sourceOut');
      if (target) target.textContent = text;
    } catch (err) {
      const errTarget = document.getElementById('sourceStatus');
      if (errTarget) errTarget.textContent = `Error: ${err.message}`;
    } finally {
      btn.disabled = false;
      btn.textContent = originalLabel;
    }
  });
})();
```

### State Registry
All generated buttons read from and write to `window.__afoState` — a shared in-page state object injected by `afo-api-binder-mcp` or `afo-ui-composer-mcp`. This is how `requires` vars are resolved.

```js
window.__afoState = window.__afoState || {};
// Variables are written here by forms, loaders, and API binding code.
// Buttons read from here before firing.
```

## Audit Checks

`audit_buttons` runs the following checks per button:

| Check | Pass Condition |
|---|---|
| `exists` | DOM element with `data-btn-id="{id}"` found in page source |
| `has_handler` | A JS event listener referencing the button's `data-btn-id` is present |
| `has_loading` | `data-loading-label` attribute present (fetch buttons only) |
| `has_requires_guard` | All `requires` vars checked before fetch fires |
| `has_success_target` | `success_target` element id referenced in handler |
| `has_error_target` | `error_target` element id referenced in handler |
| `danger_has_confirm` | `confirm()` call present for `danger` variant buttons |
| `admin_has_guard` | `hidden` class or `admin_only` guard wired to passcode state |
| `target_exists` | `success_target` and `error_target` DOM ids exist in page source |

## MCP Protocol Shape

Follows **AFO Mobile MCP Protocol** (see `toolsmith-tool-inventory.md` → `afo-mobile-mcp-protocol`):

- `POST /mcp` only
- Hand-rolled JSON-RPC 2.0
- Raw Cloudflare Worker ES module — no npm, no build
- Custom domain: `buttons.agentfeedoptimization.com/mcp`
- No Cloudflare Access on `/mcp`

## Build Notes for Claude / Cloudflare Agent

1. This is a **pure code-generation Worker** — all tools take JSON spec input and return HTML/JS string output.
2. All generated code uses `window.__afoState` as the shared state bus — do not use localStorage.
3. Generated HTML uses `data-btn-id` as the audit hook — this attribute must be preserved in all output.
4. The audit tool does **static analysis** of the page HTML source string — it does not use a headless browser. Pattern-match for `data-btn-id`, event listener registration, and target element ids.
5. Deploy to `buttons.agentfeedoptimization.com` via Cloudflare custom domain.
6. No bindings required at deploy time.
